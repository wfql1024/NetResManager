## 1. 整体架构：JavaFX WebView + 前端

### 概念

NetResManager 使用 JavaFX 的 `WebView` 组件作为 UI 容器，前端代码（HTML/CSS/JS）从 classpath 加载并在内嵌浏览器中运行。这相当于在一个桌面窗口里跑了一个简化的浏览器。

```
JavaFX Stage (桌面窗口)
  └── WebView (内嵌浏览器)
       └── index.html → CSS + JS
```

### 启动流程（MainApp.java）

```java
public void start(Stage stage) {
    webView = new WebView();
    webEngine = webView.getEngine();
    webEngine.setJavaScriptEnabled(true);

    // 从 classpath 加载 HTML
    String url = getClass().getResource("/web/index.html").toExternalForm();
    webEngine.load(url);

    // 注册 JS 桥接
    JSObject window = (JSObject) webEngine.executeScript("window");
    window.setMember("javaObject", jsBridge);
}
```

### 项目中哪里用到

整个应用都基于此架构。`MainApp.java` 是入口，`index.html` 是前端根页面。

### 初学者要点

- `WebView` 不是一个完整的 Chrome/Edge 浏览器，它是 WebKit 的精简版，对 ES2020+ 语法支持有限
- 所有静态资源（HTML/CSS/JS）放在 `src/main/resources/web/` 下，Java 通过 `getResource()` 加载

## 2. 前后端通信：JS Bridge 模式

### 概念

前端 JS 调用 `window.javaObject.methodName(args)` → Java 方法被执行 → 返回 JSON 字符串。这是同步调用模式（调用时 JS 会等待 Java 返回）。

### 后端：JsBridge.java 暴露方法

```java
public class JsBridge {
    // 每个暴露给 JS 的方法以 "js" 开头
    public String jsGetAllProjects() {
        return wrap(() -> projectService.getAllProjects());
    }

    public String jsExportFiles(String filePathsJson, int projectId) {
        return wrap(() -> {
            List<String> paths = JsonUtil.fromJson(filePathsJson, ...);
            return fileOpService.exportFiles(paths, project);
        });
    }

    // 统一错误处理包装
    private String wrap(ThrowingSupplier<?> supplier) {
        try {
            return JsonUtil.successResponse(supplier.get());
        } catch (Exception e) {
            return JsonUtil.errorResponse(e.getClass().getSimpleName(), e.getMessage());
        }
    }
}
```

### JSON 信封格式

所有 Java 返回统一格式：

```json
// 成功
{"success": true, "data": { ... }}

// 失败
{"success": false, "error": {"code": "ERROR_CODE", "message": "具体信息"}}
```

### 前端：bridge.js 封装

```javascript
// bridge.js — 统一封装，处理错误和 JSON 解析
NRM.bridge = (function() {
    function call(methodName) {
        var args = Array.prototype.slice.call(arguments, 1);
        // 调用 Java 方法
        var resultJson = window.javaObject[methodName].apply(window.javaObject, args);
        // 解析 JSON 信封
        var result = JSON.parse(resultJson);
        if (!result.success) {
            NRM.ui.showError(result.error.message);
            return null;
        }
        return result.data;
    }

    return {
        getAllProjects: function() { return call('jsGetAllProjects'); },
        exportFiles: function(paths, projectId) {
            return call('jsExportFiles', JSON.stringify(paths), projectId);
        },
        // ... 更多方法
    };
})();

// 使用时：
var projects = NRM.bridge.getAllProjects();
```

### 项目中哪里用到

- **管理界面**：扫描文件 → `NRM.bridge.scanProject(id, dir)`
- **右键菜单**：导出 → `NRM.bridge.exportFiles(paths, projectId)`
- **历史界面**：撤销 → `NRM.bridge.rollbackRecord(recordId)`
- **统计界面**：获取数据 → `NRM.bridge.getExportStatsByType(...)`

### 初学者要点

- JS 调用 Java 是**同步**的，调用期间 UI 会卡住
- 复杂参数（数组、对象）需要 `JSON.stringify()` 转为字符串传递
- 每个 Java 方法必须返回 String（JSON），不能返回 void

## 3. SPA 单页应用与路由

### 概念

SPA（Single Page Application）= 一个 HTML 文件包含所有页面，通过 JS 切换显示/隐藏来模拟页面跳转。

### 页面容器（index.html）

```html
<div id="page-container" class="page-container">
    <div id="page-manage" class="page active">  <!-- 管理页 -->
        ...
    </div>
    <div id="page-history" class="page">        <!-- 历史页 -->
        ...
    </div>
    <div id="page-statistics" class="page">    <!-- 统计页 -->
        ...
    </div>
    <div id="page-settings" class="page">      <!-- 设置页 -->
        ...
    </div>
</div>
```

```css
/* 默认隐藏所有页面 */
.page { display: none; }
/* active 类控制显示 */
.page.active { display: block; }
```

### 路由器（app.js）

```javascript
NRM.router = {
    navigate: function(page) {
        NRM.state.currentPage = page;

        // 1. 隐藏所有页面
        document.querySelectorAll('.page').forEach(function(p) {
            p.classList.remove('active');
        });

        // 2. 显示目标页面
        var pageEl = document.getElementById('page-' + page);
        if (pageEl) pageEl.classList.add('active');

        // 3. 更新侧边栏高亮
        NRM.components.navSidebar.setActive(page);

        // 4. 初始化目标页面
        switch (page) {
            case 'manage':   NRM.pages.manage.init();    break;
            case 'history':  NRM.pages.history.init();   break;
            case 'statistics': NRM.pages.statistics.init(); break;
            case 'settings': NRM.pages.settings.init();  break;
        }
    }
};
```

### 侧边栏导航触发路由

```html
<li class="nav-item active" data-page="manage">
    <span class="nav-icon">📁</span>
    <span class="nav-label">管理</span>
</li>
```

```javascript
// nav-sidebar.js
document.querySelectorAll('.nav-item').forEach(function(item) {
    item.addEventListener('click', function() {
        var page = this.getAttribute('data-page');
        NRM.router.navigate(page);
    });
});
```

### 项目中哪里用到

整个应用只有一个 `index.html`，四个页面通过路由切换。

### 初学者要点

- SPA 的核心思想：用 CSS `display: none/block` 切换页面，不刷新浏览器
- `data-*` 属性是 HTML 的自定义属性，JS 中用 `getAttribute('data-xxx')` 读取
- 用 `switch` 语句为不同页面执行不同的初始化逻辑

## 4. 模块化代码组织（IIFE 模式）

### 概念

IIFE（Immediately Invoked Function Expression）= 立即执行函数。用它创建私有作用域，只暴露需要的方法。

### 基本模式

```javascript
// file-table.js
NRM.components = NRM.components || {};  // 确保命名空间存在

NRM.components.fileTable = (function() {
    'use strict';

    // ===== 私有变量（外部访问不到）=====
    var container = null;
    var currentSort = { field: 'name', asc: true };
    var allFiles = [];

    // ===== 私有函数 =====
    function init() {
        container = document.getElementById('manage-files-container');
    }

    function render(fileGroups) {
        // ... 渲染逻辑
    }

    function applySort() {
        // ... 排序逻辑
    }

    // ===== 公开 API（return 出去的部分）=====
    return {
        init: init,
        render: render,
        sortBy: sortBy
    };
})();  // ← 注意这里的 () 立即执行

// 外部调用：
NRM.components.fileTable.render(data);
// 但访问不到 allFiles、applySort 等内部变量
```

### 项目中哪里用到

所有 JS 文件都用此模式：
- `manage.js` → `NRM.pages.manage`
- `history.js` → `NRM.pages.history`
- `file-table.js` → `NRM.components.fileTable`
- `context-menu.js` → `NRM.components.contextMenu`
- `bridge.js` → `NRM.bridge`

### 命名空间层次

```
NRM                      ← 全局命名空间
├── state                ← 全局状态
├── ui                   ← UI 工具方法
├── router               ← 路由器
├── bridge               ← Java 桥接
├── pages
│   ├── manage           ← 管理页面
│   ├── history          ← 历史页面
│   ├── statistics       ← 统计页面
│   └── settings         ← 设置页面
└── components
    ├── fileTable        ← 文件表格
    ├── contextMenu      ← 右键菜单
    ├── navSidebar       ← 侧边栏导航
    ├── projectTabs      ← 项目标签页
    ├── breadcrumb       ← 面包屑导航
    └── modal            ← 模态对话框
```

### 初学者要点

- IIFE 的核心价值：**封装**。私有变量和函数不会污染全局作用域
- `'use strict'` 开启严格模式，避免一些常见的 JS 陷阱
- `NRM.components = NRM.components || {}` 确保命名空间存在后才添加属性

## 5. 全局状态管理

### 概念

把需要在多个组件间共享的数据集中放在一个对象里，方便各模块读写。

### 状态定义（app.js）

```javascript
NRM.state = {
    currentPage: 'manage',      // 当前页面
    currentProjectId: null,     // 当前选中的项目ID（null=全部）
    currentDirectory: '',       // 当前浏览的目录
    projects: [],               // 项目列表
    files: [],                  // 当前显示的文件列表
    selectedFiles: new Set(),   // 多选的文件路径集合
    lastClickedIndex: 0         // 最后点击的行索引（Shift选择用）
};
```

### 各模块读写状态

```javascript
// manage.js — 写入
NRM.state.files = groups;
NRM.state.currentDirectory = file.path;

// file-table.js — 读取和写入
if (NRM.state.selectedFiles.has(file.path)) {
    row.classList.add('selected');
}
NRM.state.selectedFiles.add(path);

// history.js — 读取
var projectId = NRM.state.currentProjectId;

// app.js 路由器 — 写入
NRM.state.currentPage = page;
NRM.state.selectedFiles.clear();
```

### 项目中哪里用到

几乎每个模块都依赖 `NRM.state`：
- **管理界面** — `selectedFiles` 跟踪用户多选
- **历史界面** — `currentProjectId` 决定显示哪个项目的记录
- **项目标签页** — 切换 `currentProjectId` 后触发刷新
- **面包屑导航** — 读取 `currentDirectory` 显示路径

### 初学者要点

- `Set` 比数组更适合存储选中项（自动去重，`has()` 查询是 O(1)）
- 状态集中管理的好处：不用在各个函数间传来传去
- 副作用：任何模块都可以修改状态，需要自己保证一致性

## 6. DOM 操作（原生 JS）

### 概念

不依赖 jQuery 或框架，直接用浏览器原生 API 操作页面元素。

### 常用操作速查

```javascript
// ===== 查找元素 =====
var el = document.getElementById('my-id');             // 按 ID 找
var els = document.querySelectorAll('.my-class');       // 按 CSS 选择器找所有
var row = document.querySelector('tr.file-row[data-path="xxx"]');  // 属性选择器

// ===== 创建元素 =====
var div = document.createElement('div');
div.className = 'file-group';                          // 设置 class
div.setAttribute('data-path', file.path);              // 设置自定义属性
div.textContent = '纯文本内容';                         // 设置文本（防 XSS）
div.innerHTML = '<span>HTML 内容</span>';              // 设置 HTML（注意安全）

// ===== 修改元素 =====
el.classList.add('active');                            // 添加 class
el.classList.remove('hidden');                         // 移除 class
el.classList.toggle('selected', true);                 // 条件切换
el.style.display = 'none';                             // 直接设样式
el.style.display = '';                                 // 清除内联样式
var path = el.getAttribute('data-path');               // 读取自定义属性

// ===== 插入/移除元素 =====
parent.appendChild(child);                             // 末尾插入
parent.insertAdjacentHTML('beforeend', htmlString);    // 插入 HTML 字符串
parent.innerHTML = '';                                 // 清空内容
el.remove();                                           // 移除元素
```

### 批量构建表格行（来自 file-table.js）

```javascript
// 用 innerHTML 字符串拼接（适合大量元素的场景）
html += '<tr class="' + rowClass + '" data-path="'
    + escapeAttr(file.path) + '" data-size="' + file.size + '">';
html += '<td class="col-check">';
html += '<input type="checkbox" class="row-checkbox" '
    + (NRM.state.selectedFiles.has(file.path) ? 'checked' : '') + '>';
html += '</td>';
html += '<td class="col-name">' + escapeHtml(file.name) + '</td>';
html += '</tr>';

// 最后一次性插入 DOM
container.insertAdjacentHTML('beforeend', html);
```

### 项目中哪里用到

所有 JS 文件都大量使用原生 DOM API，没有引入任何 DOM 操作库。

### 初学者要点

- `innerHTML` 用于设置 HTML 内容，但直接拼接用户输入可能导致 XSS 攻击 → 需要 `escapeHtml()`
- `textContent` 用于设置纯文本，自动转义，更安全
- `insertAdjacentHTML` 比 `innerHTML +=` 更高效（不会重新解析已有内容）
- `querySelector` 中用属性选择器时，反斜杠需要双转义：`[data-path="C:\\Users"]`

## 7. 事件处理

### 概念

响应用户操作（点击、右键、悬浮等）通过事件监听器。

### 基本模式

```javascript
element.addEventListener('事件类型', function(event) {
    // event.target = 触发事件的元素
    // this = 绑定事件的元素（普通函数中）
});
```

### 7.1 点击事件 + 事件委托

```javascript
// 行点击 — 切换选中
row.addEventListener('click', function(e) {
    var tag = e.target.tagName.toLowerCase();
    // 排除：点击按钮或复选框时不触发
    if (tag === 'button' || tag === 'input') return;

    // 排除：点击在操作按钮区域内
    var el = e.target;
    while (el && el !== row) {
        if (el.classList.contains('file-row-actions')) return;
        el = el.parentElement;
    }

    toggleSelection(path);
});

// 列头点击 — 排序
th.addEventListener('click', function() {
    var field = this.getAttribute('data-sort');
    sortBy(field);
});

// 复选框点击 — 阻止冒泡
cb.addEventListener('click', function(e) {
    e.stopPropagation();  // 防止触发行点击
    toggleSelection(path);
});
```

### 7.2 右键菜单事件

```javascript
row.addEventListener('contextmenu', function(e) {
    e.preventDefault();  // 阻止浏览器默认右键菜单

    // 如果右键的是未选中的行，先选中它
    if (!selectedRecords[id]) {
        selectedRecords = {};
        selectedRecords[id] = true;
    }

    // 在鼠标位置显示自定义菜单
    NRM.components.contextMenu.show(e.clientX, e.clientY, 'history');
});
```

### 7.3 批量监听模式

```javascript
// 对匹配的所有元素绑定事件
document.querySelectorAll('.sortable').forEach(function(th) {
    th.addEventListener('click', function() {
        var field = this.getAttribute('data-sort');
        sortBy(field);
    });
});
```

### 项目中哪里用到

- **管理界面**：行点击选中、列头排序、复选框、右键菜单、悬浮按钮
- **历史界面**：行点击选中、列头排序、批量操作按钮、右键菜单
- **侧边栏**：导航点击切换页面
- **项目标签页**：点击切换项目

### 初学者要点

- `e.stopPropagation()` 阻止事件冒泡到父元素
- `e.preventDefault()` 阻止浏览器默认行为（如右键菜单、链接跳转）
- 事件委托：在父元素上监听，用 `e.target` 判断实际点击的子元素，适合动态生成的元素
- `contextmenu` 事件是右键点击事件

## 8. CSS 布局：Grid + Flexbox

### 概念

现代 CSS 两大布局利器：
- **Flexbox**：一维布局（行或列方向）
- **Grid**：二维布局（行列同时）

### 8.1 Grid — 应用主布局

```css
/* app-grid: 左侧边栏 + 右侧内容区 */
.app-grid {
    display: grid;
    grid-template-columns: var(--sidebar-width) 1fr;  /* 左固定 右填充 */
    grid-template-rows: 1fr;                            /* 单行 */
    height: 100vh;
    overflow: hidden;
}

/* 设置页: 左侧导航 + 右侧表单 */
.settings-layout {
    display: grid;
    grid-template-columns: 220px 1fr;
    gap: var(--space-lg);    /* 列间距 */
    height: 100%;
}

/* 统计页: 左右两个图表 */
.stats-charts {
    display: grid;
    grid-template-columns: 1fr 1fr;    /* 各占一半 */
    gap: var(--space-lg);
}
```

### 8.2 Flexbox — 组件内布局

```css
/* 组头: 水平排列，两端对齐 */
.file-group-header {
    display: flex;
    align-items: center;      /* 垂直居中 */
    gap: var(--space-md);     /* 子元素间距 */
}

/* 子头: 面包屑 + 操作按钮 */
.sub-header {
    display: flex;
    align-items: center;
    justify-content: space-between;  /* 两端对齐 */
}

/* 文件名单元格: 名称 + 标签 + 操作按钮 */
.file-name-cell {
    display: flex;
    align-items: center;
    gap: 4px;
    overflow: hidden;          /* 超长内容隐藏 */
}

/* 文件名截断 */
.file-name-text {
    overflow: hidden;
    text-overflow: ellipsis;   /* 超出部分显示... */
    white-space: nowrap;       /* 不换行 */
    flex-shrink: 1;            /* 可以被压缩 */
}

/* 悬浮操作按钮组 */
.file-row-actions {
    display: none;             /* 默认隐藏 */
    gap: 3px;
    margin-left: auto;         /* 推到最右边 */
    flex-shrink: 0;            /* 不被压缩 */
}
```

### 项目中哪里用到

- `app-grid`（Grid）：应用整体左-右结构
- `.settings-layout`（Grid）：设置页左-右结构
- `.stats-charts`（Grid）：统计页两个图表的左-右排列
- `.sub-header`（Flexbox）：面包屑+按钮的水平排列
- `.file-group-header`（Flexbox）：分组头的水平排列
- `.file-name-cell`（Flexbox）：文件名单元格的弹性布局

### 初学者要点

- Grid 适合页面整体框架，Flexbox 适合组件内部排列
- `1fr` 在 Grid 中表示"占剩余空间"
- `margin-left: auto` 在 Flexbox 中把元素推到最右边
- `flex-shrink: 0` 防止元素被压缩变形
- `text-overflow: ellipsis` + `overflow: hidden` + `white-space: nowrap` 三件套实现文字截断

## 9. CSS 变量（自定义属性）与主题

### 概念

用 CSS 变量统一管理颜色、间距、字号，方便换主题和修改。

### 变量定义（theme.css）

```css
:root {
    /* 颜色 */
    --color-primary: #0078d4;
    --color-primary-light: #e6f4ff;
    --color-danger: #d13438;
    --color-primary: #0078d4;

    /* 背景 */
    --bg-main: #f5f5f5;
    --bg-card: #ffffff;
    --bg-sidebar: #fafafa;
```

## 11. 表格排序

### 概念

点击表头切换升序/降序排列，用箭头指示当前排序状态。

### 表头 HTML（可点击 + 箭头占位）

```html

    文件名 <span class="sort-arrow"></span>
</th>
<th class="col-type sortable" data-sort="type">
    类型 <span class="sort-arrow"></span>
</th>
<th class="col-size sortable" data-sort="size">
    大小 <span class="sort-arrow"></span>
</th>
```

### 排序逻辑（file-table.js / history.js）

```javascript
var currentSort = { field: 'name', asc: true };

// 点击列头
function sortBy(field) {
    // 同一列 → 切换方向；不同列 → 默认升序
    currentSort.asc = (currentSort.field === field) ? !currentSort.asc : true;
    currentSort.field = field;
    // 重新排序并渲染
    applySort();
    rerenderTable();
    updateSortArrows();
}

// 比较函数
function applySort() {
    allFiles.sort(function(a, b) {
        var va, vb;
        switch (currentSort.field) {
            case 'name':
                va = a.entry.name.toLowerCase();
                vb = b.entry.name.toLowerCase();
                if (a.entry.isDirectory && !b.entry.isDirectory) return -1; // 目录优先
                if (!a.entry.isDirectory && b.entry.isDirectory) return 1;
                break;
            case 'size':
                va = a.entry.size;
                vb = b.entry.size;
                break;
            case 'modified':
                va = a.entry.modified || '';
                vb = b.entry.modified || '';
                break;
        }
        if (va < vb) return currentSort.asc ? -1 : 1;
        if (va > vb) return currentSort.asc ? 1 : -1;
        return 0;
    });
}

// 更新箭头
function updateSortArrows() {
    document.querySelectorAll('.sortable').forEach(function(th) {
        var arrow = th.querySelector('.sort-arrow');
        if (th.getAttribute('data-sort') === currentSort.field) {
            arrow.textContent = currentSort.asc ? '▲' : '▼';
        } else {
            arrow.textContent = '';
        }
    });
}
```

### 项目中哪里用到

- **管理界面**：文件名、类型、大小、修改日期 四列可排序
- **历史界面**：文件名、类型、大小 三列可排序

### 初学者要点

- `data-sort` 属性告诉 JS 按什么字段排序
- `Array.sort()` 接收比较函数，返回负数/0/正数决定顺序
- 目录优先的逻辑：在 name 排序时特殊处理，目录始终排在文件前面
- 箭头只在当前排序列显示，其他列隐藏

## 16. 异步后台任务（Java → JS 回调）

### 概念

前端调用 Java 是同步的，但某些操作（如计算文件夹大小）耗时长。方案：前端发起请求 → Java 后台线程执行 → 完成后通过 `webEngine.executeScript()` 回调 JS。

### 完整流程

```
1. JS:  NRM.bridge.startFolderSizeCalculation(paths)
2. Java: FileScanService 提交到线程池
3. Java: 后台线程递归计算每个文件夹大小
4. Java: 计算完成 → scriptExecutor.accept("NRM.onFolderSize(...)")
5. Java: Platform.runLater → webEngine.executeScript(script)
6. JS:  NRM.onFolderSize(path, size, formatted) 更新 DOM
```

### Java 侧 — 线程池 + 回调

```java
// FileScanService.java
private final ExecutorService folderSizeExecutor = Executors.newFixedThreadPool(4);
private Consumer<String> onFolderSizeResult;  // JS 回调函数

public void startAsyncFolderSizeCalculation(List<String> folderPaths) {
    for (String pathStr : folderPaths) {
        Future<?> future = folderSizeExecutor.submit(() -> {
            long size = calculateFolderSizeRecursive(Path.of(pathStr));
            String formatted = formatSize(size);

            // 构造 JS 调用字符串
            String escaped = pathStr.replace("\\", "\\\\").replace("'", "\\'");
            onFolderSizeResult.accept(
                "NRM.onFolderSize('" + escaped + "'," + size + ",'" + formatted + "')");
        });
        pendingCalculations.put(pathStr, future);
    }
}

// 取消所有挂起任务
public void cancelFolderSizeCalculations() {
    for (Future<?> f : pendingCalculations.values()) {
        f.cancel(true);
    }
    pendingCalculations.clear();
}
```

### 连接 Java 和 WebView（MainApp.java）

```java
private void registerJsBridge() {
    JSObject window = (JSObject) webEngine.executeScript("window");
    window.setMember("javaObject", jsBridge);

    // 把 executeScript 方法传给 JsBridge
    jsBridge.setScriptExecutor(this::executeScript);
}

public void executeScript(String script) {
    Platform.runLater(() -> {
        webEngine.executeScript(script);
    });
}
```

### JS 侧 — 回调函数 + 取消机制

```javascript
// app.js

// Java 完成后调用的回调
NRM.onFolderSize = function(path, size, sizeFormatted) {
    var escapedPath = path.replace(/\\/g, '\\\\');
    var row = document.querySelector(
        '#manage-files-container tr.file-row[data-path="' + escapedPath + '"]');
    if (row) {
        row.querySelector('.col-size').textContent = sizeFormatted;
        row.setAttribute('data-size', size);
    }
};

// 发起异步计算
NRM.requestFolderSizes = function() {
    NRM.cancelFolderSizes();  // 先取消旧的
    var folderRows = document.querySelectorAll(
        '#manage-files-container tr.file-row[data-isdir="true"]');
    var paths = [];
    folderRows.forEach(function(row) {
        paths.push(row.getAttribute('data-path'));
    });
    if (paths.length > 0) {
        NRM.bridge.startFolderSizeCalculation(paths);
    }
};

// 取消
NRM.cancelFolderSizes = function() {
    NRM.bridge.cancelFolderSizeCalculations();
};
```

### 项目中哪里用到

- **管理界面**：文件夹大小异步计算（先显示"计算中..."，后台算好后更新为实际大小）
- 用户点击刷新时取消所有挂起的计算任务

### 初学者要点

- JavaFX 中 `webEngine.executeScript()` 必须在 JavaFX Application Thread 上执行 → `Platform.runLater()`
- 用 `Future.cancel(true)` 中断后台线程（需在线程中配合 `Thread.interrupted()` 检查）
- JS 中的 `data-*` 属性非常适合存储元数据（如 `data-isdir="true"`, `data-size="12345"`）
- 这种模式可以推广到任何需要异步完成的 Java 操作

## 18. JavaFX WebView 兼容性注意事项

### 18.1 不支持 ES2020+ 语法

JavaFX WebView（基于 WebKit）不支持以下现代 JS 语法：

```javascript
// ❌ 可选链 — 不支持
var el = document.getElementById('id')?.classList;

// ✅ 必须用传统写法
var el = document.getElementById('id');
if (el) { el.classList.add('active'); }

// ❌ 空值合并 — 不支持
var val = input ?? 'default';

// ✅ 必须用传统写法
var val = (input !== null && input !== undefined) ? input : 'default';

// ✅ var 而非 let/const — 建议用 var（let/const 部分支持但不稳定）
var count = 0;   // 推荐
```

### 18.2 onclick 中 ID 加引号

```html

<!-- ✅ 必须加引号 -->
<button onclick="fn('1234567890-abc-def-1')">
```
